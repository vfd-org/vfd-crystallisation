## Method (native, class-number-1 definite method)

Because `h(I)=1`, the right-ideal classes of the Eichler order at the norm-31
level (the engine's kernel ideal `sigma(5phi-2)`; the reference curve's
conductor is its conjugate `(5phi-2)`) are the orbits of `A_5 = I^1/{±1}` on `P^1(O_K/N) = P^1(F_31)` via the
explicit local splitting `B ⊗ F_31 ≅ M_2(F_31)`. The Hecke operator `T_P`
(`P ∤ N`) is built from the icosian elements of reduced norm `varpi_P` (a totally
positive generator of `P`), enumerated completely by Fincke–Pohst short-vector
enumeration of the rank-8 trace-form lattice, pushed through the splitting and
accumulated on the two orbits. No Magma / Sage Brandt module is used (Sage 9.0
has no maximal order or Brandt module over `Q(sqrt5)` — see "tooling" below); the
construction is native `Z[phi]` quaternion arithmetic.

## Evidence (all parameter-free)

1. **Dimension / mass.** `A_5` acts on the 32 points of `P^1(F_31)` with exactly
   two orbits, sizes `12` and `20`. Eichler mass `sum_i 1/w_i = 1/5 + 1/3 = 8/15`
   equals the predicted `|zeta_K(-1)|/2 · (N(N)+1) = (1/60)·32 = 8/15`. The 120
   units map into `SL_2(F_31)` (reduced norm 1). Icosian trace-form Gram
   determinant `= 5^4 = 625`.

2. **Enumerator anchor.** `#{ x ∈ I : nrd(x) = varpi_P } = 120·(N(P)+1)` for every
   prime tested (the icosian Hurwitz identity) — certifies the short-vector
   enumeration is complete, with no reference to the target.

3. **Eisenstein channel.** Every `B(P)` has row sums `N(P)+1`: the Eisenstein
   eigenform `(1,1)` with eigenvalue `N(P)+1` (the bookkeeping channel) is exact.

4. **Cuspidal channel.** Every `B(P)` is self-adjoint in the orbit-size measure
   `mu = (20,12)` (`mu_I B_IJ = mu_J B_JI`) and shares the **single** cuspidal
   eigenvector `(3,-5)` across all 24 primes — a genuine simultaneous Hecke
   eigenform (the norm-31 newform), orthogonal to Eisenstein under `mu`. Its
   eigenvalue `a_P = trace B(P) − (N(P)+1)` equals the brute-force point-counted
   `a_P`:
   - **44/44 per-ideal** (superseding the original 24-bucket protocol below):
     after fixing the single labeling convention once — the engine's splitting
     root is 13, so its level is `sigma(5phi-2)` and the geometric value at
     ideal `P` is the Brandt eigenvalue at a generator of `sigma(P)` — every
     individually-labeled good prime ideal of norm ≤ 200 matches
     (`data/level31_per_ideal.json`; the script fails if any single ideal
     needs its own adjustment). The involution is the embedding `K -> F_31`
     fixed once in the splitting, exactly like the one global sign convention
     in any Hecke normalization — not a per-prime degree of freedom, and
     nothing is tuned to the target.
   - Original protocol record: 24/24 at the rational-prime level (11 in-sample,
     13 out-of-sample), see `results.json`.

## Non-fitting / non-LMFDB compliance

- Target `a_P` from brute-force point counting only (`point_count_target.py`);
  curve equation is the sole external datum; **no LMFDB eigenvalue table**.
- Geometry path (`ok_arithmetic`, `quaternion_order`, `ideal_classes`,
  `brandt_matrices`) imports none of: the target module, the curve invariants,
  LMFDB, or any regression/least-squares/curve-fit. Enforced by
  `no_fit_guard.py` (static scan) and `tests/test_no_fit_guard.py`.
- No linear map is fitted; no sign/scale/offset/phase/weight is tuned. The orbit
  weights `(20,12)` and eigenform `(3,-5)` are computed from geometry, then the
  comparison happens once.

## Tooling note (why native, not Sage/Magma)

```
Sage 9.0:  QuaternionAlgebra(K,-1,-1).maximal_order()
           -> NotImplementedError: maximal order only for rational quaternion algebras
           BrandtModule over a number field ideal -> TypeError
           HilbertModularForms -> not defined
Magma:     not available
```

So the Brandt module over `Q(sqrt5)` is built natively (no silent fallback to
Brandt modules over `Q`). Sage is used only as an independent cross-check during
development; the shipped pipeline is pure Python (`numpy` for one Cholesky).

## What this does NOT establish

A pass establishes **geometric encoding** of the cuspidal arithmetic object: the
icosian correspondence structure carries the newform and its `a_P`. It does
**not** prove RH. The remaining wall is positivity / self-adjointness of the
trace form (the Weil–Connes positivity step), which is not addressed here.
