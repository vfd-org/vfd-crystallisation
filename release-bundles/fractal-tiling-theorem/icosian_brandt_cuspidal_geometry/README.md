# Icosian Brandt Cuspidal Geometry

**WO-VFD-ICOSIAN-BRANDT-CUSPIDAL-GEOMETRY-001** — a non-fitted test of whether
cuspidal arithmetic is generated by the icosian geometric correspondence
structure over `Z[phi]`.

## What this asks

The earlier audit established that the level-1 icosian theta channel is
Eisenstein/bookkeeping only: it reproduces `zeta_K(s)·zeta_K(s-1)`, the
divisor/indexing layer, but **not** the genuine cuspidal Hecke eigenvalues of
the norm-31 elliptic curve / Hilbert newform over `K = Q(sqrt 5)`.

> The Eisenstein/theta layer shows that icosian geometry generates the bulk
> arithmetic bookkeeping. This work order tests the stronger claim: whether the
> cuspidal arithmetic coefficients arise from the nontrivial geometric
> correspondence structure of an icosian Eichler order.
>
> A pass would not prove RH. It would show that the RH-bearing arithmetic
> object is encoded by the geometry. The remaining RH wall would still be
> positivity / self-adjointness / trace-form control.

## Result

**VERDICT: `PASS_GEOMETRIC_CUSPIDAL_ENCODING`** (see `results.json`,
`WORK_ORDER_RESULT.md`).

The icosian geometry reproduces the genuine cuspidal Hecke eigenvalues `a_P`,
out of sample, with **zero fitted parameters**:

| step | what the geometry gives | check |
|------|-------------------------|-------|
| dimension | Brandt-module dim `h = 2` from the `A_5 = I^1/{±1}` action on `P^1(F_31)` | `= 2`, Eichler mass `8/15` exact |
| Eisenstein | eigenform `(1,1)`, eigenvalue `N(P)+1` | exact, every prime |
| **cuspidal** | eigenform `(3,-5)` (the **same** vector for every prime — a genuine simultaneous Hecke eigenform), eigenvalue `a_P = trace B(P) − (N(P)+1)` | **= point-counted `a_P`**, 24/24 primes (11 in-sample, 13 out-of-sample) |

The Brandt matrices `B(P)` are self-adjoint in the natural orbit-size measure
`mu = (20,12)`, and `(3,-5)` is orthogonal to the Eisenstein line `(1,1)` under
`mu` (`20·3 − 12·5 = 0`). At the primes with an unambiguous eigenvalue (inert,
ramified, and split primes whose two conjugates coincide) the match is **exact**
(7/7: `N(P) = 4,5,9,29,41,49,169`); at the remaining split primes it is exact up
to the single global Galois involution of `Q(sqrt 5)`, which permutes the two
primes above `p`.

## How it stays honest (no fitting, no LMFDB)

- **Target `a_P`** (`route_b/point_count_target.py`): brute-force point counting
  of `E(O_K/P)` for curve `31.1-a1`. The only external datum is the curve's
  Weierstrass equation (five field elements); **no LMFDB eigenvalue table**.
  Validated by Ramanujan `|a_P| ≤ 2 sqrt(N(P))` and `8 | #E(F_P)`.
- **Geometry side** (`ok_arithmetic`, `quaternion_order`, `ideal_classes`,
  `brandt_matrices`): the icosian ring, its splitting `B ⊗ F_31 ≅ M_2(F_31)`,
  the `A_5` orbits, and the Brandt/Hecke operators built from icosian elements
  of reduced norm `varpi_P`. **No curve / modular data enters this path.**
- The two sides meet only in `hecke_compare.py`, at comparison.
- `no_fit_guard.py` statically scans the geometry path and refuses any
  regression / least-squares / curve-fit / target-table reference.
- Enumerator correctness is anchored parameter-free by
  `#{ x ∈ I : nrd(x)=varpi_P } = 120·(N(P)+1)` (the icosian analogue of the
  Hurwitz identity), not by the target.

## What this does NOT do

It does **not** prove the Riemann Hypothesis. It shows the RH-bearing arithmetic
object (the cuspidal newform / its `a_P`) is **encoded by** the icosian
geometry. RH still requires the separate positivity / self-adjointness /
trace-form (Weil–Connes) step, which is untouched here.

## Layout

```
  ideal_classes.py        Phase 4  splitting, A_5 orbits, dimension gate, mass

```

## Run

```bash
python3 route_b/brandt_level31.py        # full pipeline -> results.json
python3 route_b/ideal_classes.py         # dimension gate h=2 + mass 8/15
python3 route_b/brandt_matrices.py       # Brandt matrices + common eigenform
python3 route_b/hecke_compare.py         # cuspidal comparison + verdict (slow)
for t in tests/test_*.py; do python3 "$t"; done
```

Pure Python (uses `numpy` for one Cholesky; `sympy` is **not** required at run
time). The full comparison sweep to `N(P) ≤ 200` takes a few minutes (the large
primes enumerate ~24000 icosian elements each).
