# Data

## Format

All processed `H(z)` files use one CSV with header:

```
z,H,sigma_H,source
```

- `z` вЂ” redshift (dimensionless)
- `H` вЂ” H(z) in km sвЃ»В№ MpcвЃ»В№
- `sigma_H` вЂ” 1Пѓ uncertainty in km sвЃ»В№ MpcвЃ»В№
- `source` вЂ” short reference key (e.g. `Moresco_2012`)

## Subfolders

- `raw/` вЂ” public compilations as downloaded.
- `processed/` вЂ” uniform CSVs the pipeline reads.

## Datasets to drop in

Recommended public compilations (the pipeline does **not** ship these
values to avoid any chance of mis-transcription):

- **Cosmic chronometers**: Moresco et al. compilations (2012, 2016, 2022)
  вЂ” H(z) from differential ages of passively evolving galaxies.
- **BAO-derived H(z)**: SDSS / BOSS / eBOSS / DESI DR1 line-of-sight BAO.

When you place a CSV in `processed/`, point `data_loader.load_hz` at it
and the pipeline picks it up automatically.
