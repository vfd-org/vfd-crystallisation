# VFD Explorer v1.4 — Scope

**Status:** Active (opened 2026-04-17, follows v1.3 same day)
**Theme:** Export and presentation. v1.3 gave reviewers tools to
poke the framework; v1.4 gives them polished artefacts they can
actually quote, embed, and cite.

## Why these three

The reviewer experience currently has three rough edges:

1. **The docs are text-heavy.** No plots. A 600-cell framework with
   20+ predictions deserves a "one image that summarises the case."
2. **The registry is Python-only.** A collaborator who doesn't use
   Python can't ingest the predictions for their own analysis.
3. **Paper-writing is manual.** Every new paper copy-pastes the
   same table of predictions from `vfd verify` output.

v1.4 fixes all three in one release.

## 1. Plots

Generate static SVGs (and PNG fallbacks) at build time. Embed in
the mkdocs site and in a new `docs/figures.md` page.

**Plot targets (minimal useful set):**

- `spectrum_histogram.svg` — 600-cell Laplacian eigenvalue
  multiplicities. Bars at {0, 2.3, 5.5, 9, 12, 14, 14.5, 15, 15.7},
  annotated with multiplicities, highlighting the integer sector
  {9, 12, 14, 15}.
- `vfd_vs_experiment.svg` — scatter plot, y = VFD value, x = PDG
  value, log-log for cross-particle scales. Points colour-coded by
  category (radii, masses, couplings, structural). Diagonal line
  at y = x.
- `relative_error_bars.svg` — horizontal bar chart, one bar per
  prediction, log-scale x-axis for rel. err %, tolerance marked
  as vertical threshold.
- `form_factor_curves.svg` — G_E(Q²) for VFD 120-vertex, VFD P_3,
  and the dipole fit across 0 ≤ Q² ≤ 1.5 GeV². Shows where they
  agree (low Q²) and diverge (mid/high Q²).
- `adversarial_shell_scatter.svg` — scatter of (n_min, n_max) for
  every candidate support tested; passing candidates highlighted.
  Makes "unique canonical shell" visually obvious.

**Implementation:**
- New module `src/vfd_core/figures/` with one file per plot family.
- CLI: `vfd figures --out docs/figures/` generates all plots.
- CI Pages workflow calls `vfd figures` before `mkdocs build`.

## 2. JSON / machine-readable export

**CLI:**
```bash
vfd verify --json                 # emit all results as JSON
vfd verify --json -o results.json
vfd registry --dump               # emit the registry itself as JSON
```

**JSON schema (abbreviated):**
```json
{
  "generated_at": "2026-04-17T…Z",
  "vfd_core_version": "0.2.0",
  "results": [
    {
      "id": "proton_charge_radius",
      "name": "Proton charge radius r_p",
      "vfd_value": 0.8412356,
      "unit": "fm",
      "method": "r_p = Tr(L(P_3)) × λ̄_p = 4 × λ̄_p",
      "paper_ref": "WO-PROTON-RADIUS",
      "experimental": {
        "value": 0.8409, "uncertainty": 0.0004,
        "source": "PDG 2022", "unit": "fm"
      },
      "relative_error_percent": 0.0399,
      "sigma_margin": 0.84,
      "tolerance_percent": 0.1,
      "passed": true,
      "notes": ""
    },
    …
  ]
}
```

**Implementation:**
- `validation/json_export.py` with `results_to_json_dict()`.
- `vfd verify --json` path short-circuits the pretty printer.
- `vfd registry --dump` walks `predictions.yaml` and dependencies.

## 3. LaTeX table export

**CLI:**
```bash
vfd report --format latex -o paper-results.tex
vfd report --format markdown         # still the default
vfd report --format json -o results.json
```

**LaTeX output (booktabs style):**
```latex
\begin{table}[htbp]
\centering
\caption{VFD predictions vs PDG/CODATA 2022.}
\begin{tabular}{lllll}
\toprule
ID & Prediction & VFD & Experiment & Rel. err \\
\midrule
proton\_charge\_radius & $r_p$ & $0.8412$ fm & $0.8409 \pm 0.0004$ fm & $0.04\%$ \\
…
\bottomrule
\end{tabular}
\end{table}
```

**Implementation:**
- Extend `validation/report.py` with `render_latex_report()` and
  `render_json_report()`.
- Add `--format` to `vfd report`. Default remains `markdown`.
- Escape LaTeX special characters (`_`, `%`, `$`) in IDs and notes.

## 4. Non-goals

- **No new predictions.** Still 23.
- **No interactive plots.** Static SVG/PNG only. Plotly / JupyterLite
  stays deferred.
- **No embedded HTML widgets.** The site is reproducible without JS.
- **No CSV export.** JSON covers the machine-ingest need; CSV is a
  one-liner for any user via `jq`.
- **No dynamic figure regeneration on every `vfd verify`.** Figures
  generated on demand or in CI, not by default.

## 5. Definition of done

v1.4 ships when:

1. `vfd figures --out docs/figures/` generates at least 4 SVGs
   (spectrum, VFD-vs-experiment, relative-error bars, form factor).
2. Those figures are embedded in a new `docs/figures.md` page and
   referenced from the home page.
3. `vfd verify --json` produces valid JSON matching the schema above.
4. `vfd registry --dump` produces valid JSON.
5. `vfd report --format latex` produces a compilable `\begin{tabular}`.
6. Tests cover JSON schema compliance and LaTeX escape correctness.
7. CI generates figures as part of the Pages workflow; site builds
   with figures present.
8. All existing tests still pass; `vfd verify` remains 23/23 pass.

## 6. Execution order

1. **JSON + LaTeX renderers** in `validation/report.py` + CLI flags.
2. **Figures module** + `vfd figures` CLI + one representative plot end-to-end.
3. **Remaining plots.**
4. **Embed figures in docs.** New `docs/figures.md`, add to nav, link from home.
5. **CI Pages workflow** runs `vfd figures` before `mkdocs build`.
6. **Tests + smoke + memory update.**

## 7. Risk register

- **Matplotlib in CI.** Already a dep, but headless rendering needs
  `matplotlib.use("Agg")` and no GUI backend. Figures module must
  import-safe without a display.
- **LaTeX escaping.** Characters like `_` and `%` appear in IDs and
  notes; un-escaped they break LaTeX. Use a single `_escape_latex`
  helper.
- **Figure determinism.** Plots use matplotlib defaults; set a fixed
  style and RNG seed where stochastic (none in v1.4, but future
  adversarial plots may need it).
- **File size.** SVG vector output stays small; PNG fallback capped
  at 150 DPI.
- **Plot maintenance.** Adding a prediction shouldn't break figure
  generation — use the registry as the input source so new entries
  flow through automatically.
