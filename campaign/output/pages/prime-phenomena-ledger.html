Ten prime phenomena, one instrument, one command

A ten-row ledger runs the named prime-distribution phenomena through one computational instrument: twin primes (0.003% vs Hardy–Littlewood), all even gaps, Goldbach per-N, Dirichlet equidistribution, Chebyshev bias (Leech's 26,861 exact), Q(√5) splitting, Sato–Tate on our own Brandt data, Cramér gaps (record 220 after 47,326,693 reproduced), Mersenne (the marked boundary row), and GUE zero statistics — 10/10 PASS against pre-set gates, in about ten seconds, with a falsifier stored per row.

Everything that goes in

  1. A prime sieve to 5×10⁷ and 1,983 PARI-computed zeta zeros — nothing else
  2. Per-place admissibility counting (the local layer of every row)
  3. Three theorem-grade anchor rows (Dirichlet, Chebotarev, Sato–Tate) calibrating the instrument
  4. Gates set before the data were inspected; one boundary row (Mersenne) marking where the method fails

Run it yourself

git clone https://github.com/vfd-org/closure-positivity-lab
cd release-bundles/closure-positivity-lab
python3 -m lab.prime_ledger   # deps: numpy (PARI/GP only to regenerate the zeros)

How to kill this claim

python3 -m lab.prime_ledger re-runs everything; any row drifting from its gate at larger sieve bounds kills that row, and a phenomenon that cannot be factored as (local product) × (zero-distribution statement) breaks the organising thesis — by the ledger's rules it would be published as a counterexample row.
